## What this is

This Storybook is the single source of truth for **Tenfore Fox** — a golf club booking and commerce
product — and the design system it stands on. It spans four layers: the **foundations** (design tokens),
a full **component library**, the **product experiences** assembled from them, and a **theming system** that
re-skins the whole thing per club. Everything below is live and interactive in the sidebar.

## The product — Tenfore Fox

The **Tenfore Fox** category is the assembled product: a member books a tee time, shops the **Pro Shop**,
reserves at the **Restaurant**, and completes **Checkout** to a **Confirmation**. It's the proof that the
component library and booking molecules compose into a real, branded end-to-end experience.

## The booking engine

**Booking** holds the reusable machinery — the search bar, date/rate calendars, tee-time slot cards, and
player selectors as **Molecules**, then the full **Pages** that arrange them. These same parts power the
Tenfore Fox tee-time flow and every club style guide.

## Multi-club theming & Design Systems

A single helper, `clubBrandStyle(color)`, re-points the brand namespace tokens so an entire screen adopts a
club's color with no per-component edits. The **Design Systems** category showcases this:

- **Colorways** — the full booking style guide rendered in each hue of the utility palette.
- **Color Theory** — analogous, complementary, and triadic harmonies derived from real OKLCH hue, across every chromatic base.
- **Kettle Hills · Sagamore · FloGolf** — the live per-club colorways (blue, near-black, green).
- **Accessibility** — every brand surface uses the `-700` shade and clears **WCAG AA (4.5:1)**; each guide badges its worst-case contrast.

Start at **Design Systems → Overview** for a deeper tour of that work.

## Foundations & components

**Foundations** documents the tokens everything is built from — the semantic color system (`text-primary`,
`bg-brand-solid`, `fg-success-primary`, …), the type scale, spacing, radius, and the brand logos for each
golf course. **Base** and **Application Components** are the Untitled UI library those tokens dress.

## How it's built

- **React 19** + **TypeScript** throughout
- **Tailwind CSS v4** with a semantic, light/dark-aware token system (`src/styles/theme.css`, `palette.css`)
- **React Aria Components** as the accessibility foundation (imported as `Aria*`)
- **Storybook 10** (`@storybook/nextjs-vite`) for documentation and visual testing

### Conventions

- Files are **kebab-case**; React Aria imports are prefixed **`Aria*`**.
- Style with **semantic tokens** (`text-secondary`, `bg-primary`) — never literal color classes.
- Per-club accent colors live in the theming helper, not in component code.
